前言

在学习和开发过程中，零散的代码片段、配置记录和排错经验很容易被遗忘。相比只把笔记保存在本地，我更希望搭建一个可以长期维护、方便检索、也能够对外展示的技术博客。

最终我选择了 Hexo + GitHub Pages + Butterfly 这一套方案：

  • Hexo：生成静态博客，部署简单，生态成熟；
  • GitHub Pages：免费托管静态网页，适合个人技术博客；
  • Butterfly：基于 Hexo 的高可配置主题，视觉效果和功能扩展都比较完善。

这篇文章记录博客从初始化、本地预览、GitHub Pages 部署，到 Butterfly 主题配置过程中遇到的问题和解决方法。
